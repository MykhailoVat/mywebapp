## 🚀 Швидкий старт (локально)

### 1. Клонування репозиторію

```bash
git clone https://github.com/MykhailoVat/mywebapp.git
cd mywebapp
```

### 2. Встановлення залежностей

```bash
npm install
```

### 3. Ініціалізація бази даних

Створіть базу даних та виконайте міграцію:

```bash
bash /init_db/init.sh
```

### 4. Налаштування конфігурації

Створіть два файли в кореневій директорії проекту:

**`.env_app`** — налаштування застосунку:
```env
APP_PORT=<port>
APP_HOST=<host>
```

**`.env_db`** — налаштування бази даних:
```env
DB_HOST=<host>
DB_PORT=<port>
DB_USER=<user>
DB_PASSWORD=<password>
DB_NAME=<name>
```

### 5. Запуск

```bash
npm run dev-start
```

## 🔌 Ендпоінти

> Ендпоінти `/tasks` підтримують як `application/json`, так і `text/html` — залежно від заголовку `Accept`.

### Загальні

| Метод | Шлях | Опис |
|-------|------|------|
| `GET` | `/` | Список усіх доступних ендпоінтів (повертає `text/html`) |

### Задачі

| Метод | Шлях | Опис |
|-------|------|------|
| `GET` | `/tasks` | Отримати список усіх задач |
| `POST` | `/tasks` | Створити нову задачу |
| `POST` | `/tasks/:id/done` | Позначити задачу як виконану |

#### `POST /tasks` — тіло запиту

```json
{
  "title": "Назва задачі"
}
```

> ⚠️ Поле `title` є обов'язковим. Довжина: від 1 до 30 символів включно.

#### Коди відповідей

| Код   | Опис                               |
|-------|------------------------------------|
| `200` | Запит успішно оброблено            |
| `201` | Користувача успішно створено       |
| `400` | Помилка валідації                  |
| `404` | Задачу з вказаним `id` не знайдено |
| `500` | Внутрішня помилка сервера          |

---

### Health-чеки

> ⚠️ Ендпоінти `/health` доступні **лише всередині сервера** і не мають бути відкриті назовні.

| Метод | Шлях | Опис |
|-------|------|------|
| `GET` | `/health/alive` | Перевірка: чи сервер запущений. Завжди повертає `HTTP 200` |
| `GET` | `/health/ready` | Перевірка: чи БД готова. `HTTP 200` — ОК, `HTTP 500` — помилка з'єднання |

## 🖥️ Розгортання на сервері

### Вимоги

- **ОС:** Ubuntu Server 24.04.4 LTS
- **SSH:** OpenSSH (обирається під час інсталяції або встановлюється вручну)
- Системні вимоги: [ubuntu.com/server/docs](https://ubuntu.com/server/docs/reference/installation/system-requirements/)

### Підключення до сервера

```bash
ssh username@IP_ADDRESS
```

де `username` — логін користувача, створеного під час інсталяції, `IP_ADDRESS` — адреса сервера.

### Розгортання

```bash
# Перейдіть до директорії /opt
cd /opt

# Клонуйте репозиторій
sudo git clone https://github.com/MykhailoVat/mywebapp.git

# Запустіть скрипт інсталяції
sudo ./mywebapp/scripts/main.sh
```

Після завершення скрипта сервер готовий до прийому запитів.

## 🧪 Тестування

Для тестування рекомендується використовувати [Postman](https://www.postman.com/), [Insomnia](https://insomnia.rest/) або утиліту `curl`.

### Приклад з `curl`

```bash
# Отримати всі задачі
curl -i http://192.168.122.59/tasks/

# Створити задачу
curl -i -X POST http://192.168.122.59/tasks/ \
  -H "Content-Type: application/json" \
  -d '{"title": "Моя нова задача"}'

# Позначити задачу як виконану
curl -i -X POST http://192.168.122.59/tasks/1/done
```

### Рекомендації

- Перевірте всі ендпоінти з **машини-хоста**.
- Переконайтесь, що `/health` **недоступний** ззовні.
- Спробуйте перезавантажити сервер і повторіть тестування — застосунок має підніматись автоматично.
